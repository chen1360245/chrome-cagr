# 端口清理工具使用说明

## 📁 文件说明

本目录包含两个端口清理脚本，用于清理 3000-3010 端口上的进程：

### 1. `clean_ports.py` - Python 脚本（推荐）

**优点：**
- 详细的输出信息
- 显示进程名称和PID
- 更好的错误处理
- 跨平台提示

**使用方法：**
```bash
python clean_ports.py
```

### 2. `kill_ports.bat` - Windows 批处理脚本

**优点：**
- 无需Python环境
- 双击即可运行
- 简单快速

**使用方法：**
- 直接双击 `kill_ports.bat` 文件运行

## 🚀 快速开始

### 方式一：使用 Python 脚本

```bash
# 在项目根目录执行
python clean_ports.py
```

### 方式二：使用批处理脚本

```bash
# 双击运行
kill_ports.bat
```

或者在命令行执行：
```cmd
kill_ports.bat
```

## ⚠️ 注意事项

1. **权限要求**：需要管理员权限才能终止某些进程
2. **数据保存**：脚本会强制终止进程，请确保已保存重要数据
3. **仅限 Windows**：这些脚本仅适用于 Windows 系统
4. **Node 进程**：主要用于清理开发服务器的 node.exe 进程

## 🔧 常见问题

### Q: 脚本提示"终止失败"怎么办？

A: 某些系统进程需要管理员权限，请以管理员身份运行命令提示符或PowerShell。

### Q: 如何清理其他端口范围？

A: 编辑 `clean_ports.py` 文件，修改 `main()` 函数中的：
```python
clean_ports(3000, 3010)  # 修改起始和结束端口
```

### Q: 在 Linux/Mac 上如何清理端口？

A: 使用以下命令：
```bash
# 清理单个端口
lsof -ti:3000 | xargs kill -9

# 清理多个端口
for port in {3000..3010}; do lsof -ti:$port | xargs kill -9 2>/dev/null; done
```

## 📝 技术说明

**脚本工作原理：**
1. 使用 `netstat -ano` 命令查找占用端口的进程
2. 提取 LISTENING 状态的连接的 PID
3. 使用 `tasklist` 获取进程名称
4. 使用 `taskkill /F /PID` 强制终止进程

## 🔗 相关命令

```bash
# 查看所有端口占用
netstat -ano

# 查看特定端口
netstat -ano | findstr :3000

# 手动终止进程
taskkill /F /PID <进程ID>

# 查看所有 node 进程
tasklist | findstr node.exe
```
